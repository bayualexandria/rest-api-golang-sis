Backend API Sistem Informasi Siswa

Dokumentasi Teknis Integrasi Frontend & Parameter Terperinci

Base Prefix: /api

Panduan Otentikasi

Endpoint bertanda Auth Required wajib menyertakan token JWT pada komponen Header Request:
Authorization: Bearer <your_jwt_token>

Referensi Role Akses

  • Role 1: Super Admin / Admin Sistem
  • Role 2: Kepala Sekolah / Manajemen
  • Role 3: Guru / Staf Pengajar
  • Role 4: Siswa / Wali Murid

1. Authentication & Social Login (/api/auth)

POST /api/auth/login-admin
Pengecualian status_user_id != 4

Mekanisme masuk akun administrator. Sistem akan memvalidasi bahwa akun bukan bertipe ID 4 (Siswa).

// Payload Request Body (JSON) 
{\n  "username": "admin_username",\n  "password": "secret_password"\n}
// Kemungkinan Respons Server
200 OK: Akun sukses divalidasi. Mengembalikan objek user dan string accessToken.
200 OK (Belum Verifikasi): "Email belum terverifikasi, silakan cek email anda..." (Memicu sistem mengirim NotifikasiAktivasiAkunUser).
403 Forbidden: "Username belum terdaftar" / "Password yang anda masukan salah" / "User ini tidak memiliki akses login!".
POST /api/auth/login
Wajib status_user_id = 4 (Siswa)

Pintu masuk utama untuk akun Siswa. Kebalikan dari instruksi admin, rute ini mengunci klausa database secara ketat hanya menerima entitas berstatus ID 4.

// Payload Request Body (JSON) 
{\n  "username": "nis_siswa",\n  "password": "password_siswa"\n}
200 OK: Menghasilkan data JSON lengkap {"user": {...}, "accessToken": "JWT_TOKEN", "status": 200}.
403 Forbidden: Keluar jika user tersebut bukan berstatus ID 4 (Siswa) dengan pesan "User ini tidak memiliki akses login!".
GET /api/auth/verify/:email/:token

Dipanggil ketika pengguna mengklik tautan aktivasi dari kotak masuk email mereka. Memvalidasi JWT Claims serta menghapus token lama dari DB setelah sukses.

Path Variable 1: email (Format string pendaftaran)
Path Variable 2: token (String token JWT yang diverifikasi oleh sistem melalui utils.VerifyJWT)
// Respons Sukses (200 OK): 
{\n  "message": "Email berhasil diverifikasi",\n  "status": 200\n}
BLOCK Alur Siklus Lupa Sandi Akun
POST /api/auth/forgot-password

Mengirimkan instruksi ke sistem. Menghapus rekaman token lama di tabel password_reset_tokens sebelum menyisipkan data token baru. 

Body JSON: { "email": "contoh@email.com" }
GET /api/send-reset-password/:email/:token

Metode otomatis yang membuat string sandi acak baru sepanjang 12 karakter (alphanumeric via ResetPasswordNotRandomString) kemudian menyimpannya ke user terkait.

POST /api/auth/reset-password/:email/:token

Eksekusi akhir penggantian password secara manual berdasarkan input data konformasi terenkripsi.

POST /api/logout

Keluar dari aplikasi secara aman. Sistem membaca header Authorization, memotong prefix string Bearer , mencocokkannya ke database, dan melakukan operasi Hard Delete pada data token.

// Target Query GORM: DELETE FROM personal_access_tokens WHERE tokenable_id = ?

2. User Management Group (/api/user)

GET /api/user/
Auth Required

Mengambil daftar seluruh entitas user (akun login) di sistem aplikasi.

GET /api/user/:username
Role: 1, 2, 3

Mencari dan menampilkan profile akun user berdasarkan parameter username unik.

Path Param: username (string) contoh: 530475071
GET /api/user/guru/:nip
Role: 2, 3

Mengambil data user spesifik kategori Guru berdasarkan NIP (Nomor Induk Pegawai).

3. Data Siswa Management (/api/siswa)

GET /api/siswa
Role: 1, 2, 3, 4

Menampilkan seluruh data sekumpulan objek siswa yang berada di database.

PATCH /api/siswa/:nis
Role: 1, 2, 3, 4

Mengupdate data entitas siswa secara parsial berdasarkan NIS (Nomor Induk Siswa).

Skema Skrip Update (JSON)
// Body Patch (Kirim kolom yang ingin diubah saja): 
{\n  "nama_lengkap": "Budi Santoso",\n  "alamat": "Jl. Merdeka No. 12"\n}

4. Data Guru Management (/api/guru)

GET /api/guru
Role: 1, 2, 3, 4

Mengambil kumpulan data seluruh guru untuk dipetakan pada sistem informasi.

5. Sistem & Utilitas Global

GET /api/drop-email-verified-at/:username

⚠️ Endpoint Kebutuhan Debug / QA Testing

Menghapus stempel tanggal verifikasi email (email_verified_at = NULL) agar akun kembali ke status belum terverifikasi untuk disimulasikan ulang.

POST /api/logout
Role: 1, 2, 3, 4

Menghapus sesi token aktif di sisi *client side* dan mereset status otentikasi middleware.
